# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in vazr, please **do not** open a public GitHub issue. Instead, please email the maintainer directly with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if available)

**Email:** chakravarthi1597@gmail.com

We take security seriously and will respond to reports within 48 hours. We appreciate your responsible disclosure and will work with you to fix the issue before public disclosure.

## Supported Versions

Only the latest release version of vazr is actively supported for security updates. Users are encouraged to upgrade to the latest version.

| Version | Status |
|---------|--------|
| 1.1.x | ✅ Supported |
| 1.0.x | ⚠️ Limited support |
| < 1.0 | ❌ Unsupported |

## Security Updates

Security updates will be released as patch versions (e.g., 1.1.1) as soon as possible after verification and fix.

## Dependencies

We use Dependabot to automatically check for known vulnerabilities in our npm dependencies. All security updates to dependencies will be prioritized.
